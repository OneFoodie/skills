# skills
已有的所有技能支持描述

## 核心技能分类

### 1. 项目管理与规划
- **project-requirements-clarification**：项目需求澄清，通过苏格拉底式提问挖掘核心价值和目标
- **project-roadmap-planning**：项目路线图规划，基于产品概述和模块依赖规划开发顺序
- **project-tech-stack**：项目技术栈选择，推荐最合适的技术栈并生成文档
- **project-structure**：项目结构设计，基于技术栈和业务核心设计目录结构
- **planning-and-task-breakdown**：任务分解，将工作拆分为有序任务

### 2. 开发流程
- **feature-requirements-clarification**：功能需求澄清，挖掘用户意图和验收标准
- **feature-tech-design**：功能技术设计，产出包含API、数据库、核心逻辑的详细方案
- **feature-task-planning**：功能任务规划，将技术方案拆解为可执行的开发任务
- **feature-implementation**：功能编码实现，按TDD流程执行开发任务
- **feature-evolution**：功能变更管理，处理已有功能的修改和扩展需求

### 3. 代码质量与维护
- **code-review-and-quality**：代码审查，从多个维度评估代码质量
- **code-simplification**：代码简化，提高代码清晰度而不改变行为
- **bugfix-workflow**：bug修复工作流，系统化定位和修复bug
- **systematic-debugging**：系统调试，使用系统性方法进行根因分析
- **deprecation-and-migration**：废弃和迁移管理，处理旧系统的移除和用户迁移

### 4. 前端开发
- **frontend-ui-engineering**：前端UI工程，构建生产级用户界面
- **browser-testing-with-devtools**：浏览器测试，使用Chrome DevTools进行测试

### 5. 后端与架构
- **api-and-interface-design**：API和接口设计，指导稳定的API设计
- **performance-optimization**：性能优化，优化应用性能
- **security-and-hardening**：安全加固，增强代码安全性

### 6. 工具与流程
- **git-workflow-and-versioning**：Git工作流，管理代码版本和分支
- **ci-cd-and-automation**：CI/CD自动化，设置构建和部署流水线
- **mcp-builder**：MCP服务器构建，构建生产级MCP工具
- **using-git-worktrees**：Git工作树使用，创建隔离的开发环境

### 7. 文档与沟通
- **documentation-and-adrs**：文档和ADR，记录决策和文档
- **chinese-documentation**：中文技术文档写作，符合国内团队文化
- **writing-plans**：编写计划，创建详细的实现计划
- **writing-skills**：写作技能，提升文档质量

### 8. 团队协作
- **team_workflow**：团队工作流，基于任务复杂度智能调配专家资源
- **receiving-code-review**：接收代码审查，处理审查反馈
- **requesting-code-review**：请求代码审查，验证工作成果

## 技能使用流程

### 项目启动流程
1. **project-requirements-clarification**：澄清项目需求
2. **project-product-overview**：生成产品概述
3. **project-tech-stack**：选择技术栈
4. **project-structure**：设计项目结构
5. **project-roadmap-planning**：规划开发路线图

### 功能开发流程
1. **feature-requirements-clarification**：澄清功能需求
2. **feature-tech-design**：设计技术方案
3. **feature-task-planning**：规划开发任务
4. **feature-implementation**：实现功能
5. **verification-before-completion**：验证完成情况

### 问题解决流程
1. **bugfix-workflow**：处理bug
2. **systematic-debugging**：系统调试
3. **code-review-and-quality**：代码审查

## 技能特点与优势

1. **系统化流程**：每个技能都有明确的工作流程和步骤
2. **模板化输出**：提供标准化的文档模板
3. **专业领域覆盖**：涵盖软件开发全生命周期
4. **中文支持**：部分技能专门针对中文团队优化
5. **可扩展性**：支持自定义和扩展

## 应用建议

1. **根据阶段选择技能**：在项目不同阶段选择合适的技能
2. **组合使用**：多个技能可以组合使用，形成完整工作流
3. **遵循流程**：严格按照技能的工作流程执行
4. **持续改进**：根据实际使用情况不断调整和优化

通过合理利用这些技能，可以显著提高开发效率和代码质量，确保项目的顺利进行。
